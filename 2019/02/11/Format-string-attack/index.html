source code:

#include <stdio.h>

int main(int argc, char * argv[])
{
    char a[1024];
    while(1) 
    {
        memset(a, '\0', 1024);
        read(0, a, 1024);
        printf(a);
        fflush(stdout);
    }
    return 0;
}

透過執行可以知道就是一個我們輸入什麼他printf什麼

但他的printf因為是直接使用printf(a)來做輸出
因此有format string的弱點，盲測的時候可以試著輸入看看%x或其他printf的關鍵字。
只要有像這樣的回應就代表有戲(?

可以透過aaaa%n$x來找offset，n是任意數，就是透過aaaa%1$x aaaa%2$x…這樣子下去爆破取得offset
%n是代表第n個參數，$x是以hex的方式輸出，參照wiki和owasp:

For example, printf("%2$d %2$#x; %1$d %1$#x",16,17) produces 17 0x11; 16 0x10.

Below are some format parameters which can be used and their consequences:

•"%x" Read data from the stack

•"%s" Read character strings from the process' memory

•"%n" Write an integer to locations in the process' memory
%d - 十进制 - 输出十进制整数
%s - 字符串 - 从内存中读取字符串
%x - 十六进制 - 输出十六进制数
%c - 字符 - 输出字符
%p - 指针 - 指针地址
%n - 到目前为止所写的字符数

但這邊會遇到一個問題

不開PIE，x86 ELF預設都是從0x8048000開始，但%s遇到\x00就會中斷，
要怎麼解決呢?

把位置顛倒放就好了，這樣他就會先leak出內容再print address，但因為位置換了，所以我們的offset要跟著換才行

以這題來說就是%5$s+p32(address)，因為p32(address)移到後面去，而且要把%4$s這四個byte算進去，所以要往後一個offset

以上面教學網址的範例是%13$saaa+p32(address)這點讓我比較不解，後面得知的結論是，如果沒加aaa
%13$s共5byte，這樣子在他後面的address會被stack切到而讀不完整，所以要塞aaa讓他變成8個byte
讓後面的address能完整個放在一格stack上，這樣子讀stack的時候才能完整讀到address

方法都知道了，現在我們可以leak整個binary
